Блок 5

Следующий этап - «Классификация» - реализованный здесь алгоритм, предназначен для того, чтобы на основе картины кластеризации, полученной в процессе предшествующего этапа, для train-данных, которые приобрели роль «обученных данных» рассчитать классифицирующие функции принадлежности, оценивающие «вероятность» (на самом деле, конечно – лишь, некую ее косвенную характеристику) принадлежности точки данных к тому или иному из выявленных «кластеров/классов». Для демонстрации, упомянутые функции обычно рассчитывается сперва для каждой точки самого train-файла – их количество всегда равно количеству найденных кластеров.

Чтобы провести классификацию , нам из проекта, где находится train-файл , -точнее, файл содержащий разметку на кластеры ( т.е., столбец «cluster_id» или «subcluster id») , нужно выбрать файл (чисто условно называемый нами test), к которому мы хотим применить результаты этой разметки - рассчитать по ней значения функции принадлежности точек, находящихся в test - что и составляет здесь суть классификации. (в том числе, в роли test может выступать и сам train-файл с результатами, - алгоритм позволяет выбирать его). Назначив «test»-файл, мы (из подмножества общих для TRAIN и для TEST координат) должны выбрать координаты, участвующие в классификации, и снова выполнить процедуру «Подбор констант».

Для демонстрации, на примере самого train-файла упомянутые функции рассчитывается в создаваемых для этого столбцах, сперва от столбцов координат и id Кластеров для каждой точки самого train-файла – количество функций (столбцов) всегда равно количеству выявленных в процессе анализа кластеров.

Точно так же, данный набор функций может быть рассчитан для любой точки, заданной в системе, имеющей минимум одну общую координату с системой, в которой заданы точки train-данных.

Суть и возможности реализованного здесь решения, демонстрирует следующий пример открыть файл с данным для классификации здесь {‘train-данные(Х1-Х5) .xlsx ‘} : видим train-данные, описанные системой из пяти координат: Х1, Х2, Х3, Х4, Х5. Для использования в кластеризации оставлены только первые две координаты Х1 иХ2.

«Test-данные» ссылка на файл { ‘ Test-данные(Х3,5,6,7).xlsx ‘} заданы в системе Х3, Х5, Х6, Х7 - имея таким образом, лишь две общие координаты с train, (причем, среди них вообще нет ни одной из тех, что использована для кластеризации - такого ограничения метод не накладывает).

В результате выполнения кластеризации-субкластеризации мы получили из train-данных подмножества - покажем их на проекции Х1Х2 (вернемся для этого к рис.4) - то есть, именно на тех координатах, на которых и кластеризовали – картина выглядит наглядно.

А теперь покажем эти же самые train-данные на проекции Х3Х5(рис.7.1) - пока без разбиения на кластеры - взгляд на картину, наводит на предположение, что применение методов кластеризации к данным, если выбрать для анализа эти Х3 и Х5 едва ли даст продуктивный результат - данные выглядят «монолитно» и «аморфно».

Теперь на следующей иллюстрации (рис.7.2) покажем результаты кластеризации, полученные на Х1 и Х2, но спроецированные на Х3Х5. Взгляд на картинку, располагает к выводу (пусть, и умозрительному, – цель таких примеров – наглядность и не более) что в отличие от кластеризации, метод классификации (относящийся уже к категории методов «обучение с учителем») имеет все шансы оказаться уже вполне результативным.

«Учителем» здесь выступает уже полученная пред этим на других координатах разметка train- данных на кластеры. В каждой точке рассчитываются персональные классифицирующие функции принадлежности, – по числу найденных кластеров. По величине каждой такой функции в данной точке мы судим о «вероятности» верного отнесения точки к кластеру. Понятно, что значения этих функций могут быть получены, как для точек из train, так и для точек из test, - короче, вообще для любой точки, которую возможно описать в подсистеме Х3Х5.
