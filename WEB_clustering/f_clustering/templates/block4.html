Блок 4

Следующий этап «Кластеризация»- включает собственно алгоритм Кластеризации - может быть пропущен, для перехода сразу у Субкластеризации (т.е., к «конвексной» кластеризации). Выполнив кластеризацию, получаем результаты - а именно файл по длине (количеству строк т.е.) и составу id совпадающий с train- файлом, имеющий все поля координат из train-файла (только значения координат так и остаются приведенными в нем в нормированном виде) и дополнительно – справа добавляется поле «Cluster ID» - сообщающее кластерную принадлежность каждой точки из train-файла .

Алгоритм «Мерджинг» может быть опционально выполнен, после процедуры Кластеризации ( и при обязательном условии выполнения этой процедуры, естественно) - он работает с полученными кластерами и сокращает их количество, выполняя слияние кластеров, в случаях когда находит их разграничение не вполне обоснованным.

Алгоритм «Субкластеризация» (выявление covex-подмножеств) может быть выполнен после Кластеризации (или Кластеризации+Мерджинг), либо - сразу, минуя Кластеризацию;

В первом варианте, поиск covex-подмножеств выполняется уже по отдельности – внутри найдены кластеров, что сокращает объем вычислений.

По выполнении субластеризации, так же получаем результаты: а именно файл по длине (количеству строк т.е.) и составу ID совпадающий с train- файлом, имеющий все поля координат из train-файла (в нормированных значениях, там где были вычислены нормы) а так же - поле «Cluster ID» - сообщающее кластерную принадлежность каждой точки из train-файла - если этап кластеризации был пропущен, то поле «Cluster ID» не будет присутствовать. И наконец, как результат - появляется поле «СубкластерID» , заполненное идентификаторами принадлежности каждой точки к тому или иному субкластеру (convex-множеству). Результаты могут быть просмотрены непосредственно со страницы Сервиса (нажатие «Посмотреть данные» на вкладке «Кластеризация»), либо скачены:

а) в виде одного файла [ нажать «Скачать данные»] ; б) набора файлов по количеству найденных кластеров/сабкластеров [ нажать «Скачать данные по кластерам / по сабкластерам»].}
